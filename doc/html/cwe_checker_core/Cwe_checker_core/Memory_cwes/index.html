Module Cwe_checker_core.Memory_cwes

  • How the check works

This module implements memory-related CWE checks.

Right now the check detects cases of

  • CWE 415: Double Free
  • CWE 416: Use After Free

How the check works

Via Dataflow Analysis, the check tries to keep track of all memory objects and pointers known at specific points in the program. It also keeps track of the status of memory object, i.e. if they have been already freed. Access to an already freed object generates a CWE warning. In cases where the analysis cannot reliably determine whether accessed memory has been freed or not, a CWE warning may (or may not) be issued to the user based on the likelihood of it being a false positive.

Note that the check is still experimental. Bugs may occur and the rate of false positive and false negative warnings is not yet known.
